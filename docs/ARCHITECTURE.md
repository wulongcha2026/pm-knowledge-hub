## 核心模块详解

### 1. 基础知识库 (Core App)

**功能：**
- 文档 CRUD
- 分类/标签管理
- 关键字搜索（SQLite FTS5）
- 浏览统计

**数据模型：**
```python
Document (文档)
├── title: 标题
├── content: 内容
├── summary: 摘要
├── category: 分类 (FK)
├── tags: 标签 (M2M)
├── file: 附件
├── is_public: 公开标记
└── view_count: 浏览次数

Category (分类)
└── 支持层级结构 (parent FK)

Tag (标签)
└── 唯一名称

SearchLog (搜索日志)
└── 记录搜索行为
```

---

### 2. RAG 知识问答助手 (RAG App)

**工作流程：**
```
用户上传文档
    ↓
文档分块 (按段落)
    ↓
bge-m3 嵌入模型生成向量
    ↓
存储到 Chroma 向量数据库
    ↓
用户提问
    ↓
问题向量化
    ↓
Chroma 相似度检索 (Top-K)
    ↓
构建上下文 + 提示词
    ↓
GLM-4.7 生成答案
    ↓
返回答案 + 引用来源
```

**关键技术：**
- **嵌入模型**: bge-m3 (本地运行，1024 维)
- **向量数据库**: Chroma (持久化存储)
- **LLM**: 智谱 GLM-4.7 (OpenAI 兼容接口)
- **分块策略**: 按段落分割 (可优化为递归分块)

**数据模型：**
```python
VectorDocument (向量文档)
├── title: 标题
├── content: 内容片段
├── chunk_index: 分块索引
├── embedding: 向量 (JSON)
└── source_document: 来源 (FK)

Conversation (对话)
├── user: 用户 (FK)
├── title: 对话标题
└── 时间戳

Message (消息)
├── conversation: 对话 (FK)
├── role: user/assistant
├── content: 内容
└── sources: 引用来源 (JSON)
```

### 3. 项目风险识别 Agent (Risk Agent App)

**LangGraph 工作流：**
```
输入 (项目描述 + 会议纪要)
    ↓
[Node 1] 风险分类
    - 识别相关风险类别
    ↓
[Node 2] 技术风险分析
    - 技术栈风险
    - 架构风险
    - 集成风险
    ↓
[Node 3] 进度风险分析
    - 时间估算
    - 依赖关系
    - 关键路径
    ↓
[Node 4] 资源风险分析
    - 人力资源
    - 设备资源
    - 预算风险
    ↓
[Node 5] 综合评估
    - 计算整体风险等级
    - 生成风险报告
    ↓
输出 (风险清单 + 应对建议)
```

**风险类别：**
- 技术风险 (technical)
- 进度风险 (schedule)
- 资源风险 (resource)
- 需求风险 (requirement)
- 沟通风险 (communication)
- 外部风险 (external)

**风险等级计算：**
```python
风险分数 = 概率 (1-10) × 影响 (1-10)

平均分 >= 64 → critical (严重)
平均分 >= 36 → high (高)
平均分 >= 16 → medium (中)
平均分 < 16  → low (低)
```

**数据模型：**
```python
RiskAssessment (风险评估)
├── user: 用户 (FK)
├── title: 评估标题
├── project_description: 项目描述
├── meeting_notes: 会议纪要
├── overall_risk_level: 整体风险等级
├── status: 状态
└── 时间戳

RiskItem (风险项)
├── assessment: 评估 (FK)
├── category: 风险类别
├── title: 风险标题
├── description: 风险描述
├── risk_level: 风险等级
├── probability: 概率 (1-10)
├── impact: 影响 (1-10)
└── mitigation_suggestions: 应对建议
```

## API 设计

### RESTful API
